  • Notes(s):

    1. CORS: Browsers should not send credentials in preflight requests irrespective of this setting. When making credentialed requests to a different domain, third-party cookie policies will still apply. The policy is always enforced independent of any setup on the server and the client. Preflight requests must never include credentials.
    2. Response-> .status/.statusText/.ok are the most common fetch responses (status).
    3. BODY request and responses can contain body data.
    4. FETCH api support can be detected by checking for Headers, Request, and Reponse.
    5. For older browsers, use 'Polyfill' technique to support Fetch.
